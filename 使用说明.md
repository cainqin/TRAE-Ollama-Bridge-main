# TRAE-Ollama-Bridge 中文使用说明

## 这是什么？

**TRAE-Ollama-Bridge** 是一个本地桥接工具，它将你电脑上的 Ollama 模型包装成 OpenAI 格式的 API。

**解决的问题**：TRAE、Cursor 等 IDE 强制使用 OpenAI 等固定服务商，无法直接调用本地 Ollama 模型。本项目可以绕过这个限制。

---

## 一、环境准备

### 1.1 安装 Node.js

本项目基于 Node.js 运行，需要 v18 或更高版本。

**下载地址**：https://nodejs.org/ （选择 LTS 长期支持版）

**验证安装**：打开命令提示符（cmd）或 PowerShell，输入：

```bash
node --version    # 应显示 v18.x.x 或更高
npm --version     # 应显示 10.x.x 或更高
```

### 1.2 安装 Ollama 并下载模型

1. 访问 https://ollama.com/ 下载并安装 Ollama
2. 安装完成后，在终端中拉取模型，例如：
   ```bash
   ollama pull gemma3:4b
   ollama pull qwen2.5:7b
   ```
3. 确保 Ollama 服务在后台运行（托盘区有 Ollama 图标）

## 二、快速上手指南

### 2.1 首次启动

**方式 A — 自动启动（推荐）**

双击项目根目录的 `Start-Bridge.bat`，系统会自动完成：
1. 安装 Node.js 依赖（首次运行）
2. 启动桥接服务
3. 在浏览器中打开管理界面

**方式 B — 菜单交互程序**

双击项目根目录的 `TRAE-Bridge.cmd`，会显示菜单：

```
===========================================
   TRAE-Ollama-Bridge 管理程序
===========================================
  1. 启动桥接服务（前台运行）
  2. 查看服务状态
  3. 安装依赖（npm install）
  4. 查看环境配置
  5. 运行自动测试
  0. 退出
===========================================
```

### 2.2 访问 Web 管理界面

启动后，浏览器会自动打开 http://localhost:3000/，你将看到以下功能面板：

| 功能区域 | 说明 |
|:--------|:-----|
| **特权桥接服务** | 安装/卸载 Windows 服务、应用/撤销拦截策略 |
| **Ollama 模型列表** | 显示本地 Ollama 中的所有模型，一键复制名称 |
| **模型映射** | 将 Ollama 模型名映射为 OpenAI 格式 ID |
| **聊天测试** | 测试桥接是否正常工作 |

## 三、配置步骤详解

### 3.1 创建映射关系

在 WebUI 的"模型映射"区域：

1. 点击 **刷新列表** 查看当前映射
2. 点击 **+ 新增映射**
3. 填写：
   - **本地模型名称**：如 `gemma3:4b`（从"Ollama 模型列表"复制）
   - **映射 ID**：自定义名称，如 `OpenAI-gemma3:4b`（将在 IDE 中使用）
4. 点击 **保存**

### 3.2 验证桥接是否正常

在 WebUI 的"聊天测试"区域：

1. **映射 ID**：选择刚才创建的映射（如 `OpenAI-gemma3:4b`）
2. **是否流式**：建议选择"流式"
3. **测试模式**：选择"显式桥接（本地 /v1）"
4. **输入内容**：在下方输入框中打字，点击 **发送**
5. 如果对话框显示模型回复，说明桥接成功！

### 3.3 配置透明拦截（可选）

如果需要让 IDE 直接访问 `https://api.openai.com` 并自动转到本地模型：

1. 在 WebUI 中点击 **注册并启动服务**（首次需 UAC 确认）
2. 点击 **应用拦截策略**（安装 CA 证书、写入 hosts、配置端口转发）
3. 点击 **系统状态** 确认显示：
   ```
   HTTPS：已启用 · hosts：已写入
   ```
4. 在"聊天测试"中将模式切换为"透明拦截"再次测试

### 3.4 配置 TRAE IDE

1. 确认桥接服务已启动并在聊天测试中验证通过
2. 打开 TRAE IDE，点击 AI 对话框的 **设置图标 → 模型 → 添加模型**
3. 填写：
   - **服务商**：选择 `OpenAI`
   - **模型**：选择 `自定义模型`
   - **模型 ID**：填写 WebUI 中设置的映射 ID（如 `OpenAI-gemma3:4b`）
   - **API 密钥**：可填任意值（默认宽松模式）
4. 点击 **添加模型**
5. 回到 AI 聊天框，选择刚添加的模型即可使用

## 四、配置文件说明

项目根目录的 `.env` 文件（需自行从 `.env.example` 复制）控制所有行为：

| 变量 | 默认值 | 说明 |
|:-----|:-------|:-----|
| `PORT` | `3000` | 桥接服务端口 |
| `BIND_ADDRESS` | `127.0.0.1` | 监听地址（`127.0.0.1` 仅本地，`0.0.0.0` 开放网络） |
| `HTTPS_ENABLED` | `false` | 是否启用 HTTPS |
| `OLLAMA_BASE_URL` | `http://127.0.0.1:11434` | Ollama 服务地址 |
| `ACCEPT_ANY_API_KEY` | `true` | `true`=接受任意密钥，`false`=强制校验 |
| `EXPECTED_API_KEY` | 空 | 固定密钥值（`ACCEPT_ANY_API_KEY=false` 时生效） |
| `STRIP_THINK_TAGS` | `false` | `true`=自动移除 `<think>` 推理标签 |
| `ELEVATOR_PORT` | `55055` | 高权限助手端口 |

## 五、两种使用模式

### 模式 1：显式桥接（推荐新手）

IDE 中配置自定义 Base URL 为 `http://localhost:3000/v1`。

**优点**：无需管理员权限，配置简单
**适用**：支持自定义 API Base 的 IDE 或客户端

### 模式 2：透明拦截（高级）

系统级劫持 `api.openai.com` 域名，将其指向本地桥接服务。

**优点**：适用于强制使用 OpenAI 且不可修改 Base URL 的 IDE
**缺点**：需要管理员权限（UAC 确认），配置步骤较多

## 六、常见问题

### Q1: 启动时报 "端口被占用"

**原因**：`PORT=3000` 已被其他程序占用。

**解决**：修改 `.env` 中的 `PORT` 为其他值（如 `3001`），然后重启。

### Q2: "显式桥接"聊天成功，但 IDE 无法连接

**检查项**：
1. IDE 的 Base URL 是否设置为 `http://localhost:3000/v1`
2. 如果 IDE 强制 HTTPS，需先启用 `HTTPS_ENABLED=true`
3. 部分 IDE 要求模型 ID 以 `gpt-` 开头，可将映射 ID 设为 `gpt-3.5-turbo` 尝试

### Q3: "透明拦截"模式下聊天失败

**检查顺序**：
1. 在 WebUI 点击 **系统状态**，确认显示 `HTTPS：已启用 · hosts：已写入`
2. 以管理员身份运行 PowerShell，执行：
   ```powershell
   netsh interface portproxy show all
   ```
   应能看到 `443 → 127.0.0.1:3000` 的转发规则
3. 检查 hosts 文件（`C:\Windows\System32\drivers\etc\hosts`）中是否有：
   ```
   127.0.0.1 api.openai.com
   ```

### Q4: 返回内容包含 `<think>...</think>` 标签

**解决**：将 `.env` 中的 `STRIP_THINK_TAGS=true`，重启服务即可自动过滤。

### Q5: 如何撤销透明拦截？

在 WebUI 中：
1. 点击 **撤销拦截策略**（清理 hosts 和端口转发）
2. 点击 **卸载服务**（移除 Windows 服务）
